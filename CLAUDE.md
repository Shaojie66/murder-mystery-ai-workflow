This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## 项目概述

剧本杀AI创作工作流——一个人用AI工具链完成6人剧本杀的完整创作，涵盖从灵感到商业化发布的完整生命周期。

```
阶段1：机制设计（2周）
  └── Gemini深度研究 / 规则书改编

  └── 内测反馈 + 迭代修复

阶段5：商业化（1周）← 新增
  └── 成本核算 + 定价策略 + 销售渠道

总周期：14-16周
```

## 核心问题：六视角信息差管理

剧本杀区别于普通小说的关键：每个角色有自己的背景故事，在群体事件中各自看到不同版本，但不能穿帮。

**解决方案**：
1. 先建信息矩阵表：每个事件 × 每个角色 = 该角色知道什么
2. AI生成文本后，逐段对照信息矩阵检查
3. 用"盲区设计"（传言/模糊处理）代替错误信息

详见：`docs/02_剧本创作.md`

## 关键约束

- 剧本杀固定为6人
- 每人有a本（阅读本）+ b本（线索本）
- 必须避免穿帮：角色A不能知道角色B当时不知道的信息
- 遵循8阶段工作流：机制设计→剧本创作→视觉物料→用户测试→商业化→印刷生产→宣发内容→社区运营

## 执行工作流

1. **复制模板**：`templates/` 下的模板可直接复制使用
2. **创建项目**：新建项目时，复制 `项目执行清单.md` 作为起点
3. **记录执行**：在 `logs/` 下记录执行过程和遇到的问题
4. **更新文档**：变更时更新 `CHANGELOG.md`
