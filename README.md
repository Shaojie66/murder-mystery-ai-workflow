## 核心问题：六视角信息差

剧本杀区别于普通小说的关键：**信息差设计**

```
角色A视角：看到事件X的版本1
角色B视角：看到事件X的版本2
角色C视角：只看到事件X的部分片段
...

六人拼图 = 完整真相
```

**穿帮场景（必须避免）**：
- 角色A的本里提到了角色B当时不知道的信息
- 某个角色知道的事情违背了时间线逻辑
- 结局揭示的信息与前面某角色本里的描述矛盾

**解决方案**：
1. 先建信息矩阵表：每个事件 × 每个角色 = 该角色知道什么
2. AI生成文本后，逐段对照信息矩阵检查
3. 用"盲区设计"故意留白，而不是给错误信息

## 创作检查清单

### 阶段1-3：核心创作
- [ ] 机制确定（原创 or 改编）
- [ ] 六角色定位分配（参考：武将/谋士/文官/刺客/医者/隐者）
- [ ] 核心事件线梳理（3-5个关键事件）
- [ ] 信息矩阵表建立
- [ ] 每人独立背景故事
- [ ] 每人a本（阅读本）+ b本（线索本）
- [ ] AI生成初稿
- [ ] 人工信息差审核
- [ ] 文风统一润色
- [ ] 视觉物料生成

### 阶段4-6：商业化准备 ⭐新增
- [ ] 用户测试（5-8人）
- [ ] P0问题修复
- [ ] 成本核算
- [ ] 定价策略
- [ ] 销售渠道搭建
- [ ] 印刷打样
- [ ] 量产确认

### 阶段7-8：发布与运营 ⭐新增
- [ ] 宣发内容制作
- [ ] 多平台发布（B站/小红书/公众号/抖音）
- [ ] 社群搭建
- [ ] 反馈收集
- [ ] 扩展包规划
