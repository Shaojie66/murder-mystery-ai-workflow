## 风险管理

### 风险矩阵

| 风险 | 概率 | 影响 | 应对 |
|------|------|------|------|
| 剧本穿帮未发现 | 高 | 高 | 多轮测试、穿帮检测AI |
| 印刷质量不达标 | 中 | 高 | 先打样、多家比价 |
| 首批印刷积压 | 中 | 中 | 首批少印、预售验证 |
| 宣发效果差 | 中 | 中 | 多平台矩阵、持续优化 |
| 玩家反馈差 | 低 | 高 | 测试阶段充分验证 |

### 风险应对策略

```
高概率+高影响：必须预防
中概率+高影响：必须预案
低概率+高影响：应急预案
```

## 团队协作（单人+AI）

### 角色分工

| 角色 | 负责人 | 职责 |
|------|--------|------|
| 总负责 | 创作者 | 决策、质量把控 |
| 机制设计 | Gemini AI | 深度研究、规则分析 |
| 剧本创作 | Claude/ChatGPT | 大纲、扩写、润色 |
| 视觉物料 | 即梦AI | 人物图、海报、卡牌 |
| 质量检测 | Claude | 穿帮检测、一致性检查 |
| 宣发运营 | 创作者+AI | 内容创作、发布 |

### AI工具调度

```
项目启动 → Gemini（机制研究）
　　　↓
剧本创作 → Claude（大纲+扩写+穿帮检测）
　　　↓
视觉物料 → 即梦（批量生成）
　　　↓
质量检测 → Claude（一致性检查）
　　　↓
宣发内容 → Claude/ChatGPT（文案+优化）
```

## 质量管理

### 各阶段质量标准

| 阶段 | 质量标准 | 检查方式 |
|------|----------|----------|
| 机制设计 | 机制可行、平衡 | 原型测试 |
| 剧本创作 | 无穿帮、逻辑自洽 | 穿帮检测AI |
| 视觉物料 | 风格统一、清晰 | 对比审核 |
| 用户测试 | 无P0问题 | 测试报告 |
| 印刷生产 | 颜色准确、尺寸一致 | 抽检 |
| 宣发内容 | 数据达标（曝光/转化） | 数据复盘 |
