# 04_用户测试

> 在正式投产前发现剧本问题，确保玩家体验。

---

## 目的

用户测试是量产前最后一次系统性问题发现机会。通过真实玩家游戏，发现：
- 穿帮/逻辑漏洞
- 角色平衡性问题
- 游戏节奏问题
- 体验不佳的环节

---

## 测试流程

### 1. 招募测试玩家

| 要求 | 说明 |
|------|------|
| 人数 | 5-8人（含DM） |
| 经验 | 至少玩过1次剧本杀 |
| 角色分配 | 随机分配或按意愿 |

**招募渠道**：
- 朋友圈/微博征集
- 剧本杀群组
- 线下店玩家群

### 2. 测试前准备

- [ ] 打印测试版剧本（可以黑白打印）
- [ ] 准备角色牌/线索卡
- [ ] 设定测试时长（建议3-4小时）
- [ ] 准备反馈问卷

### 3. 完整游戏测试

按照正式游戏流程进行：
1. DM开场介绍
2. 分发角色本
3. 阅读a本（第一幕）
4. 自我介绍
5. 搜证/讨论
6. 读取b本（第二幕）
7. 指认凶手
8. 复盘

### 4. 收集反馈

**即时反馈（游戏结束后）**：
- 哪些地方让你困惑？
- 哪个角色你觉得最弱/最强？
- 哪些线索你觉得没用/太少/太多？
- 节奏哪里太慢/太快？

**问卷反馈**：
- 角色体验评分（1-10）
- 剧情吸引力评分（1-10）
- 整体满意度（1-10）
- 开放式建议

---

## 问题分类与优先级

### P0（必须修复）

- 穿帮：角色本内信息与事实矛盾
- 逻辑漏洞：关键证据链断裂
- 死局：无法推理出正确答案

### P1（建议修复）

- 角色不平衡：某角色明显太弱/太强
- 节奏问题：某个环节拖沓
- 线索问题：某些线索无效或重复

### P2（可选优化）

- 文笔润色
- UI/排版优化
- 体验细节提升

---

## 迭代修复

| 轮次 | 内容 | 产出 |
|------|------|------|
| 第1轮 | 完整测试 + 收集反馈 | 反馈汇总 |
| 第2轮 | 修复P0问题 | 修复版剧本 |
| 第3轮 | 快速复测（1小时） | 验证修复 |
| 定稿 | 综合评审 | 最终版 |

---

## 关键检查点

```
□ 所有角色本时间线一致
□ 每个人知道的信息符合设定
□ 关键证据指向唯一凶手
□ 线索数量适中（每人3-5条）
□ 游戏时长合理（3-4小时）
□ 所有结局逻辑自洽
```
