# 阶段二：剧本创作

## 核心挑战：六视角信息差管理

这是剧本杀创作中最容易穿帮、也最体现功力的部分。

---

## 信息矩阵法

### 第一步：建立事件-角色矩阵

```markdown
# 信息矩阵表

| 事件 | 角色A | 角色B | 角色C | 角色D | 角色E | 角色F |
|------|-------|-------|-------|-------|-------|-------|
| 事件1 | 知道全部 | 知道全部 | 只知部分 | 不知道 | 知道全部 | 知道部分 |
| 事件2 | 知道版本A | 知道版本B | 知道版本A | 不知道 | 知道版本A | 知道版本B |
| ... | ... | ... | ... | ... | ... | ... |

# 图例
- 知道全部：该角色完整经历此事
- 知道部分：只经历此事的一部分
- 知道版本X：经历此事但认知被误导/不完整
- 不知道：此事发生时该角色不在场/无感知
```

### 第二步：盲区设计原则

```
故意留白 > 错误信息

❌ 穿帮写法：角色A写道"当时B明明在场"，但B的剧本里他当时不在
✅ 正确写法：A可以"听到有人说B来过"（模糊处理）

❌ 穿帮写法：C知道所有角色的秘密
✅ 正确写法：C只知道他自己的秘密，可以通过猜测试图揭露他人

✅ 技巧：使用"传言"、"据说"、"我怀疑"来给不确定信息
```

## 六角色结构设计

### 每人需要两本

```
a本 = 阅读本（游戏前阅读）
├── 背景故事（这个角色的过去）
├── 核心秘密（这个角色隐藏什么）
├── 事件记忆（这个角色经历的事件）
└── 任务目标（游戏中要达成什么）

b本 = 线索本（游戏中阅读）
├── 支线剧情
├── 隐藏技能触发条件
└── 角色专属结局触发
```

### 角色类型模板

| 类型 | 特点 | 信息量 | 穿帮风险 |
|------|------|--------|----------|
| 核心角色 | 主线推动者 | 高 | 中 |
| 辅助角色 | 提供线索 | 中 | 低 |
| 隐藏角色 | 身份特殊 | 高 | 高 |
| 死者角色 | 剧本中死亡 | - | - |

## AI辅助写作五步流程

```
第一步：写大纲（人工）
   ↓
第二步：分章节（人+AI）
   ↓  同时建立信息矩阵
第三步：AI生成初稿
   ↓
第四步：人工审核 + 信息差检查
   ↓
第五步：AI润色定稿
```

### 详细指令

#### 第一步：人工写大纲

```markdown
# [角色名] 大纲

## 核心冲突
[这个角色内心最矛盾的是什么]

## 关键转折点
- 转折1：[事件] → [角色反应]
- 转折2：[事件] → [角色反应]

## 结局走向
[角色最终的命运/归属]

## 与其他角色的纠葛
- 角色B：[关系描述]
- 角色C：[关系描述]
```

#### 第三步：AI生成指令

```
请根据以下大纲，写[角色名]的a本第一章：

角色设定：[粘贴角色设定]
章节大纲：[粘贴章节大纲]
信息矩阵：[粘贴该角色在此事件中知道什么]

写作要求：
- 字数：3000-5000字
- 风格：[古风/现代/奇幻]
- 叙事视角：第一人称
- 注意：只写该角色视角，不要泄露其他角色当时不知道的信息
- 禁止出现：该角色不可能知道的细节

请开始写作。
```

#### 第四步：人工审核检查清单

```
□ 角色性格一致（语气/用词/行为是否像这个人）
□ 信息差合规（对照信息矩阵，该角色知道的才能写）
□ 时间线正确（事件发生顺序是否自洽）
□ 叙事节奏（是否太拖沓/太跳跃）
□ 伏笔埋设（是否有呼应后续的线索）
□ 穿帮检查（其他角色的秘密是否被无意泄露）
```

## 穿帮案例与修复

### 案例1：时间线穿帮

```
❌ 原问题：
角色A写道"那天晚上我看到B从C的房间出来"

✓ 但B的剧本里，那天晚上B在别处

✅ 修复：
改为"我似乎看到一个人影从C的房间出来"（不确定是谁）
或改为"A听说B那天晚上去过C的房间"（传言而非亲眼）
```

### 案例2：身份穿帮

```
❌ 原问题：
角色A写道"我知道D是卧底"

✓ 但D的剧本里，卧底身份要到第三幕才揭示

✅ 修复：
改为"A怀疑D，但不确定"或"A听到传言说D可疑"
```

### 案例3：能力穿帮

```
❌ 原问题：
角色E写道"E拥有读取他人内心的能力"

✓ 这能力在机制里不存在

✅ 修复：
删除此描述，或修改为"E善于察言观色"（游戏外描述）
```
