# expand 操作规范

## 目的

`expand` 将原型剧本（2人 + 3事件）扩写为完整剧本（6人 + 5-7事件）。

## 触发条件

- 必须处于原型模式（`state.is_prototype == True`）
- 必须存在 `mechanism.md` 和 `characters.md`
- expand 后 `is_prototype` 置为 `False`

## 扩展单位

| 字段 | 操作 | 说明 |
|------|------|------|
| `outline` | 保留 | 原型大纲不变，作为扩写基准 |
| `mechanism` | 保留 | 核心机制不变（事件数从3扩到5-7） |
| `characters` | **替换** | 新生成6人版本，原版角色为锚定 |
| `information_matrix` | **替换** | 同步更新 6×(5-7) 矩阵 |

### 扩展矩阵

```
原型角色（2人）──锚定──► 完整角色（6人）
     │                      │
     │              ├── 新角色A（与原型角色1关联）
     │              ├── 新角色B（与原型角色2关联）
     │              ├── 新角色C（新增第三方势力）
     │              └── 新角色D（可选，对立阵营）
     │
     ▼
扩展后事件线（3条──►5-7条）
     │
     ├── 新增事件：原有角色在新事件中的新视角
     ├── 新增事件：仅新角色知道的秘密
     └── 新增事件：连接新旧角色的关键交叉点
```

## 保留字段（不可变）

以下字段 expand 后保持不变，用户明确选择覆盖时才修改：

1. **核心机制**：`mechanism.md` 中的机制规则
2. **故事主题**：outline 中的核心冲突和结局走向
3. **原型角色**的性格、背景、秘密（在新版本中继承）
4. 时代背景（era/time setting）

## 可扩展字段

以下字段在 expand 时由 LLM 重新生成：

1. **角色剧本**（characters.md）
   - 保留原型角色的性格和背景（约束）
   - 新角色须与原型角色逻辑自洽
   - 新角色的动机须与核心冲突挂钩

2. **信息矩阵**（information_matrix.md）
   - 从 2×3 扩展为 6×(5-7)
   - 原型角色在新事件中的信息更新
   - 新角色从其视角独立构建

3. **事件列表**（从 mechanism.md 提取）
   - 从 3 条扩展为 5-7 条
   - 保持原型 3 条事件顺序不变
   - 新事件插入或延续原有事件链

## 幂等性

**不保证幂等**。expand 是创作性操作，每次调用 LLM 可能生成不同内容。

- 第一次 expand：基于原型生成完整版
- 第二次 expand：基于当前完整版重新生成（可能引入新变化）

如需确定性结果，第二次 expand 前应从备份文件恢复。

## 文件操作

```
expand 前：
  characters.md          ← 原型角色（备份为 characters.md.proto.{timestamp}）
  information_matrix.md ← 原型矩阵（备份为 information_matrix.md.proto.{timestamp}）

expand 后：
  characters.md         ← 新生成 6 人版本
  information_matrix.md ← 新生成 6×(5-7) 矩阵
```

备份文件保留于项目目录，供需要时手动恢复。

## 依赖传播

expand 后以下内容**不会**自动更新（需用户手动触发）：

| 内容 | 更新方式 |
|------|---------|
| `test_guide.md` | 需重新运行阶段4生成 |
| `image-prompts.md` | 需重新运行阶段3生成 |
| `commercial.md` | 需重新运行阶段5生成 |
| `script.pdf` | 需重新运行阶段6生成 |

## expand 的限制

以下情况 expand **可能无法**良好工作：

1. **原型角色与其他角色深度耦合**：如果原型角色A的秘密依赖于尚未创建的角色B，则扩写时可能穿帮
2. **机制有阵营限制**：如果机制固定为2阵营，扩写到6人需要额外设计
3. **原型只有单条事件线**：3条事件线太少，无法为6人提供足够的信息差

## 使用约束

```python
# 仅在原型模式下可 expand
if not state.is_prototype:
    console.print("[yellow]当前不是原型，跳过 expand[/yellow]")
    return False
```

## 扩展检查清单（expand 后人工确认）

- [ ] 新角色与原型角色无逻辑矛盾
- [ ] 6人视角覆盖完整事件链
- [ ] 无角色知道其不应该知道的信息（穿帮检查）
- [ ] 核心机制未被改变
- [ ] 结局走向与原型大纲一致
