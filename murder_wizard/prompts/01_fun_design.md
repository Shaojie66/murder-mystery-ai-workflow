# 欢乐设计 Prompt — 欢乐本

> 适用于 欢乐本（fun）类型剧本杀
> 核心：欢乐氛围、社交互动、游戏性

---

## 欢乐本设计原则

1. **社交即产品** — 剧本的核心是让玩家互动、碰撞、产生笑点
2. **人设即笑点** — 每个角色本身就是喜剧担当
3. **反转即惊喜** — 预期反差是欢乐本的核心武器
4. **轻松即正义** — 不能让玩家"累"，要享受游戏过程

---

## Q1：欢乐基因验证

```
【brief.md 核心内容】
{{brief_content}}

在设计任何具体内容之前，先回答：

1. 这个剧本的**欢乐核心**是什么？
   - 主要笑点类型是什么？（误会/反转/夸张/讽刺/自嘲...）
   - 笑点是**设计出来的**还是**自然产生的**？
   - 玩家是否能**主动创造笑点**而不只是被动接受？

2. **社交活跃度验证**：
   - {{char_count}}个角色之间是否有足够的**社交碰撞点**？
   - 是否有"**破冰机制**"让陌生玩家快速熟络？
   - 是否存在**阵营/对抗**来激发社交互动？

3. 最危险的欢乐设计失误：
   - 是否会**尴尬**（让玩家觉得在尬演）？
   - 是否有**低俗梗**（不是为了笑而是恶心）？
   - 欢乐是否**喧宾夺主**（忘了推理/情感主线）？

请给出：
- 欢乐基因评分（1-10，附理由）
- 3个以内**必须解决**的欢乐设计问题
```

## Q3：游戏性验证

```
请验证以下欢乐设计的游戏性：

【完整欢乐设计】
{{mechanism_content}}

验证维度：

### 社交活跃度验证
- 是否有足够的**社交机会**让所有玩家参与？
- 是否有"**社交死角**"（某角色被孤立）？
- 是否需要**强制社交机制**（如必须和某人对话才能获得线索）？

### 笑点密度验证
- 开场是否有**足够的笑点**来破冰？
- 中盘是否有**笑点升级**（比开场更好笑）？
- 结局前是否有**收尾笑点**（让玩家笑着结束）？

### 节奏验证
- 笑点和推理/情感的比例如何？
- 是否有**喘息空间**（不需要一直笑）？
- DM的**控场难度**如何？（不能让游戏失控）

### 防尴尬验证
- 是否会**过度表演**（玩家觉得尴尬）？
- 是否有**低俗/敏感内容**？
- 玩家是否可以**选择参与程度**（不想演可以不演）？

对于每个问题，给出：
- 问题描述
- 严重程度（P1/P2/P3）
- 具体修复建议
```

## Q4：阵营/对抗设计

```
基于以下已完成的欢乐设计，设计阵营或对抗方案：

【完整欢乐设计 + 游戏性报告】
{{mechanism_and_balance}}

阵营设计要求：
- 阵营必须有**利益冲突**（不是表面划分而是真实对抗）
- 阵营必须有**喜剧性**（对抗本身要好笑）
- 玩家必须有**选择权**（可以不选边站）

阵营类型参考：
- 对抗型：两队对立，如：甲方vs乙方
- 混战型：所有人互怼，如：谁是内鬼
- 合作型：联合对抗某个目标，如：一起找出真凶
- 间谍型：隐藏身份，如：谁是大反派

每个阵营方案包含：
- 阵营划分
- 各阵营的**喜剧目标**
- 阵营间的**互动规则**
- 胜负判定方式

格式：每个方案一节，300字以内
```
