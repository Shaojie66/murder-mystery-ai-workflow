# 推理设计 Prompt — 推理本

> 适用于 推理本（reasoning）类型剧本杀
> 核心：逻辑推理、线索链、公平性验证

---

## 推理本设计原则

1. **逻辑自洽即产品** — 线索链必须形成唯一闭合解
2. **公平即尊严** — 玩家获取线索的途径必须对等
3. **悬念即驱动力** — 每个谜题必须引发下一个谜题
4. **成就感即奖励** — 破解真相时必须有强烈的满足感

---

## Q1：推理框架验证

```
【brief.md 核心内容】
{{brief_content}}

在设计任何具体线索链之前，先回答：

1. 这个剧本的**推理核心**是什么？
   - 核心谜题是什么？（不是"谁杀了谁"，是"为什么这么设计"）
   - 破解核心谜题需要哪些**推理能力**？（演绎/归纳/排除/重组...）
   - 玩家是否有**足够的信息**来破解？（不能靠运气）

2. **公平性验证**：
   - 线索是否对所有玩家**同等可及**？
   - 是否存在"**唯一信息源**"（某个线索只有一个人能提供）？
   - 凶手是否有**不合理的信息优势**？

3. 最危险的推理设计失误：
   - 是否有"**伪线索**"（看起来有关但实际无关）？
   - 是否有"**循环论证**"（用A证B，用B证A）？
   - 线索是否有过多的**随机性**？

请给出：
- 推理框架评分（1-10，附理由）
- 3个以内**必须解决**的推理设计问题
```

## Q3：公平性验证

```
请验证以下推理设计的公平性：

【完整推理设计】
{{mechanism_content}}

验证维度：

### 信息可及性验证
- 每个玩家是否有**同等的信息获取机会**？
- 是否存在"**信息孤岛**"（某个角色只能通过一个人获取关键信息）？
- 凶手的信息优势是否**合理且有限**？

### 逻辑链完整性
- 每条线索链是否**无断裂**？
- 是否存在"**缺环**"（需要但找不到的线索）？
- 玩家是否可以**反向验证**（从结论推回线索）？

### 难度曲线验证
- 开场是否有**足够的推理空间**？（不能只是读本）
- 中盘是否有**推理升级**？（从简单线索到复杂组合）
- 结局前是否有**关键推理点**？（让玩家感觉突破在即）

### 防杠验证
- 是否有"**显然但错误**"的推理路径？（干扰项）
- 是否有"**永远无法排除**"的嫌疑人？
- DM是否有足够的**引导话术**来帮助玩家？

对于每个问题，给出：
- 问题描述
- 严重程度（P1/P2/P3）
- 具体修复建议
```

## Q4：凶手设计

```
基于以下已完成的推理设计，为{{char_count}}位玩家设计凶手方案：

【完整推理设计 + 公平性报告】
{{mechanism_and_balance}}

凶手设计要求：
- **凶手必须有合理的存在理由**（不是"为了当凶手而存在"）
- **凶手的获胜策略必须可行**（不能是靠玩家失误获胜）
- **凶手的弱点必须存在但隐蔽**（高手能发现但需要努力）

每个凶手方案包含：
- 凶手是谁（哪个角色）
- 凶手的**核心策略**（游戏中如何获胜）
- 凶手的**致命弱点**（玩家如何识别）
- 凶手中盘后的**应对方案**（如果玩家已经怀疑）

格式：每个方案一节，300字以内
```
