## 背景

- **现有产品**: murder-wizard CLI 工具，8 阶段剧本杀创作工作流
- **当前状态**: Python CLI，本地执行，已有种子用户

## 实验详情

### 实验 1：订阅付费 Fake Door

- **假设**: 用户愿意为云端功能订阅付费
- **方法**: 创建"Pro 版本即将上线"落地页，展示定价（¥29/月），放置"立即订阅"按钮（点击后显示"功能开发中，优先体验请留邮箱"）
- **指标**: 点击率、留邮箱转化率
- **成功阈值**: 点击率 > 5%，留邮箱转化 > 20%
- **决策标准**:
  - 达标 → 继续做付费功能
  - 不达标 → 重新思考定价或功能定位

### 实验 2：引导式 vs 技术向 A/B 测试

- **假设**: 门槛降低能带来用户增长
- **方法**: 制作两个落地页：A = 当前 CLI 介绍（技术向），B = 简化引导式介绍（新手友好）。在相同渠道投放，对比转化率
- **指标**: 落地页 → 注册转化率
- **成功阈值**: B 版本转化率比 A 高 30% 以上
- **决策标准**:
  - B 胜出 → 以引导式为核心卖点
  - A 胜出 → 维持技术向定位

### 实验 3：协作需求用户访谈

- **假设**: 创作者有强烈协作需求
- **方法**: 招募 10 名剧本杀创作者，访谈 + 问卷
- **指标**: 有协作经历比例、痛点出现频次、付费意愿评分
- **成功阈值**: > 50% 用户有协作经历，且视为痛点
- **决策标准**:
  - 协作是真实痛点 → 优先做协作功能
  - 反之 → 搁置或降低优先级

### 实验 4：引导式原型可用性测试

- **假设**: 引导式 UI 能兼顾新手和高级用户
- **方法**: 制作引导式创建流程原型，邀请 5 名新手 + 5 名高级用户，完成"创建剧本"核心任务
- **指标**: 任务完成率、完成时间、NASA-TLX 认知负荷评分
- **成功阈值**: 两组任务完成率均 > 80%，认知负荷评分差异 < 20%
- **决策标准**:
  - 两组都满意 → 引导式方案可行
  - 新手满意但高级用户不满 → 考虑"专家模式"开关

### 实验 5：可视化首次点击测试

- **假设**: 创作者想要可视化而非纯文本
- **方法**: 展示两个界面 Mockup：A = 纯文本编辑器，B = 可视化故事板图。记录用户首次点击
- **指标**: 选择可视化用户的比例
- **成功阈值**: > 60% 用户选择可视化界面
- **决策标准**:
  - 达标 → 可视化作为核心交互
  - 不达标 → 文本编辑器优先，可视化为辅助

### 实验 6：AI 穿帮检测技术验证

- **假设**: AI 穿帮检测准确率高
- **方法**: 用现有 CLI 输出的剧本样本，运行穿帮检测脚本，人工评估误报率和漏报率
- **指标**: 准确率 = (正确识别数) / (总问题数)
- **成功阈值**: 准确率 > 85%，误报率 < 10%
- **决策标准**:
  - 达标 → 作为 Pro 功能核心卖点
  - 不达标 → 降低优先级，先做规则引擎辅助

## A/B 测试路由说明

| URL | 用途 |
|-----|------|
| `/landing` | 50/50 随机分配，基于 userAgent 哈希保证会话一致 |
| `/landing?variant=a` | 强制显示 A 版本（技术向） |
| `/landing?variant=b` | 强制显示 B 版本（新手友好） |

部署后可在各广告渠道使用不同 URL，追踪转化率差异。
