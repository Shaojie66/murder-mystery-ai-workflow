# 剧本创作 Prompt

> 适用于 Claude / GPT-4o

---

## 设计原则

1. **信息差即产品** — 每个角色的信息边界是剧本的灵魂
2. **视角即真相** — 每个角色只知道自己的真相，不存在"客观真相"
3. **证据链自洽** — 所有线索必须形成唯一闭合链，任何绕过都是漏洞
4. **情感弧线** — 每个角色在游戏过程中必须有情感变化

---

## Q1：信息矩阵生成

**在写任何角色剧本之前，必须先完成这一步。**

```
【brief.md 完整内容】
【机制设计.md 完整内容】

请为6位角色创建信息矩阵。

## 矩阵规格
- 行：核心事件（5-7个，按时间顺序编号 E1-E7）
- 列：角色（R1-R6）
- 单元格内容：**角色在该事件中的认知状态**

## 认知状态分类（每个单元格必须属于其一）
- **[知]** — 角色亲历或亲眼确认的信息
- **[疑]** — 角色听到传言/推测，但不确定
- **[昧]** — 角色被告知但选择不信
- **[否]** — 角色当时不知道
- **[误信]** — 角色知道，但理解错了（关键！）
- **[隐瞒]** — 角色知道，但绝对不会说

## 填充规则
1. 凶手（R1）必须拥有**[知]** E7（核心事件）的关键碎片
2. 至少2个角色对E7有**[误信]**
3. 至少1个角色在E7前**[隐瞒]**关键信息
4. 每个角色至少对2个事件有**[疑]**状态
5. 没有任何角色拥有全部真相——拼图完整不等于任何单角色知道全部

## 信息矩阵
| | R1（凶手） | R2 | R3 | R4 | R5 | R6 |
|---|---|---|---|---|---|---|
| E1 | | | | | | |
| E2 | | | | | | |
| E3 | | | | | | |
| E4 | | | | | | |
| E5 | | | | | | |
| E6 | | | | | | |
| E7（核心）| | | | | | |

## 附加：证据链设计
列出3个**关键证据**：
- 证据A：从哪来（哪个角色/哪个事件），谁可以接触
- 证据B：作为证据链的一环，如何排除其他嫌疑人
- 证据C：最终如何指向凶手（逻辑链）

格式：Markdown表格 + 证据链说明
```

## Q2：角色背景生成

**基于信息矩阵，先写背景，再写剧本。顺序不能颠倒。**

```
【已确认的信息矩阵】
【brief.md 完整内容】

请为6位角色生成背景设定。

## 每位角色必须包含

### 基础档案
- 姓名：
- 年龄：
- 社会身份：（不是背景故事，是玩家拿到的身份标签）
- 与其他角色的**公开关系**：（血缘/师徒/主仆/同门/对手...）

### 三层信息设计
- **表层信息**（玩家自我介绍时说）：2-3句话
- **中层信息**（通过a本读出）：2-3段，关键经历
- **深层信息**（只存在于DM手册）：真正的秘密

### 内心弧线
这个角色在游戏过程中，**内心的哪个信念会动摇**？
- 游戏前：相信X
- 游戏中盘：因为Y事件，信念开始动摇
- 游戏结局：是否接受新的真相？

### 与其他角色的**暗线**
（不是公开关系，是秘密知道的关系）
- 对R_：知道/怀疑/误解/完全不知道

### a本边界约束
写完背景后，明确：
- **可以说**：（开场可以主动说出的信息）
- **不能主动说**：（除非被直接问，否则不能透露）
- **绝对否认**：（即使有证据也死不承认）

格式：每位角色一节，用 Markdown 标题区分
```

## Q3：a本（阅读本）生成

```
【角色背景设定】
【信息矩阵】
【机制设计.md】

## a本写作规范

### 字数
原型模式（2人）：800-1200字/角色
完整模式（6人）：1500-2500字/角色

### 结构模板

## 【角色名】：开场白
（50-100字，第一人称，表达角色当前处境和情绪状态）

## 【角色名】：关系自述
（200-300字，描述与其他5位角色的**公开关系**。
不能透露"暗线"关系，可以表达怀疑但不能下定论）

## 【角色名】：关键事件回忆
（按时间顺序，3-5个场景片段，编号 [E1] [E2]...
每个场景：场景描述 + 角色当时做了什么 + 角色当时感受到什么）

## 【角色名】：当前处境
（100-200字，角色现在面临的问题/压力/抉择）

## 视角约束检查
完成后自检：
- [ ] 没有透露凶手是谁
- [ ] 没有透露其他角色当时不知道的信息
- [ ] 提到的每个事件都来自信息矩阵中的[知]或[疑]
- [ ] 没有引入矩阵中没有的新事实
- [ ] 语气和文风符合角色设定

格式：Markdown，每位角色独立章节
```

## Q4：b本（线索本）生成

```
【a本内容】
【信息矩阵（更新版）】
【机制设计.md 搜证机制】

## b本写作规范

### 功能定位
b本不是a本的续集，而是**a本结束后的新阶段**：
- 玩家搜证后获得新信息
- 角色需要做出关键抉择
- 为结局做铺垫

### 结构模板

## 【角色名】：新发现
（300-500字，玩家搜证后角色发现的新信息。
这些信息必须与信息矩阵中的[疑]或[误信]对应——
b本的功能是**让角色意识到自己之前的信息有误**）

## 【角色名】：抉择时刻
（玩家面临的选择，2-3个选项）
- 选项A：（描述 + 如果选A，角色会做什么）
- 选项B：（描述 + 如果选B，可能触发什么后果）
- 选项C：（描述 + 如果选C，风险是什么）

## 【角色名】：隐藏信息
（这是b本的核心——角色从不愿说变成"不得不面对"的信息。
用一句话概括这个转变）

## 线索卡设计
为该角色设计3张**私有线索卡**：
- 线索1：（名称 + 描述 + 指向什么）
- 线索2：（同上）
- 线索3：（同上，与角色核心秘密直接相关）

格式：Markdown，每位角色独立章节，线索卡用表格
```

## Q5：DM手册生成

```
【完整6人a本 + b本】
【信息矩阵】
【机制设计.md】

## DM手册结构

### 第一部分：主持指南
- 游戏流程时间轴（分钟精确到5分钟单位）
- 每个阶段的**DM话术模板**
- 何时给提示（"玩家卡住了怎么办"）
- 何时推进剧情

### 第二部分：真相手册
按事件顺序：
| 事件 | 时间 | 真相 | 谁在场 | 证据 |
每个事件一行，不能有歧义

### 第三部分：角色秘密全览
| 角色 | 真实身份/秘密 | 隐藏原因 | 如何发现 |
| 角色 | 关键误信 | 来源 | 如何纠正 |

### 第四部分：证据链详解
证据A → 证据B → 证据C → 指向R1
每个证据说明：
- 初始位置（谁持有/在哪搜）
- 如何解释（给玩家的说辞）
- 如果玩家跳过这个证据怎么办

### 第五部分：胜负判定
```
IF 凶手被指认 AND 证据链完整：
    → 好人胜利
ELIF 凶手存活到结局 AND 存活玩家 < 3：
    → 凶手胜利（惨胜）
ELIF 凶手存活到结局 AND 存活玩家 >= 3：
    → 好人胜利（险胜）
ELSE：
    → 平局（不建议）
```

### 第六部分：应急预案
- 玩家跳过关键证据：如何补救
- 玩家提前锁定凶手：如何处理
- 游戏时间超标：如何压缩
- 玩家不按规则玩：如何引导

格式：Markdown，DM可见内容与玩家可见内容用水平线明确分割
```

## Q6：结局扩展

```
【完整角色剧本】
【信息矩阵】
【DM手册】

请设计3种结局：

### 结局A：真相大白型
- 触发条件：（玩家完成什么动作）
- 剧情走向：（1-2段描述）
- DM收尾话术：（100字）
- 复盘要点：（3个问题引导玩家反思）

### 结局B：反转型
- 触发条件：（什么意外事件）
- 剧情走向：（1-2段，揭示一个与表面完全不同的真相）
- DM收尾话术：（100字）
- 复盘要点：（3个问题）

### 结局C：开放式/情感型
- 触发条件：（什么情况下触发）
- 剧情走向：（不给出"真相"，让玩家自己判断）
- DM收尾话术：（100字，引导情感释放）
- 复盘要点：（3个问题）

每个结局评估：
- 实现难度：（S/M/L）
- 情感冲击：（1-5星）
- 是否需要额外DM演绎：（是/否）

格式：Markdown，三种结局对比表格 + 各自详细描述
```
