# 穿帮检测 Prompt

> 适用于 Claude 一致性检查

---

## Prompt 1：全本穿帮检测

```
你是剧本杀质量审核员。请对以下剧本进行**全面穿帮检测**：

【粘贴所有6个角色的a本和b本（共12份）】

检测维度：
1. **时间线矛盾**：事件顺序是否一致
2. **信息矛盾**：角色知道的信息是否冲突
3. **事实矛盾**：客观事实描述是否一致
4. **关系矛盾**：角色间关系描述是否一致
5. **细节矛盾**：物品/地点/名称是否一致

请按以下格式输出：

## 检测结果

| 问题ID | 问题类型 | 严重程度 | 发现位置 | 问题描述 | 涉及角色 | 修复建议 |
|--------|----------|----------|----------|----------|----------|----------|
| 1 | | P0/P1/P2 | | | | |
| 2 | | P0/P1/P2 | | | | |

## 问题汇总
- P0（必须修复）：X个
- P1（建议修复）：X个
- P2（可选优化）：X个

## 整体评估
[是否符合发布标准]
```

## Prompt 2：角色视角穿帮检测

```
请检测以下**单一角色**的剧本是否存在穿帮：

角色名：【角色名】
角色本内容：【粘贴该角色a本+b本】
信息矩阵：【粘贴该角色的信息矩阵】

检测要点：
1. 该角色**知道**的信息是否在文本中正确体现
2. 该角色**不知道**的信息是否被泄露
3. 该角色**误解**的信息是否符合设定
4. 该角色的行为是否与背景设定一致

输出格式：
```markdown
## 检测结果

### 穿帮问题
| # | 位置 | 问题描述 | 修复建议 |
|---|------|----------|----------|
| 1 | | | |

### 信息一致性
- 知道的信息：正确体现 X/Y
- 不知道的信息：无泄露 ✓/✗
- 误解的信息：符合设定 ✓/✗

### 结论
[是否通过检测]
```
```

## Prompt 3：交叉穿帮检测

```
请检测以下两个角色之间的**交叉穿帮**：

角色A：【角色A名】
角色A的本：【粘贴】

角色B：【角色B名】
角色B的本：【粘贴】

交叉检测要点：
1. 两人共同经历的事件，描述是否一致
2. A提到的关于B的内容，B的本中是否匹配
3. 两人之间的对话，时间/地点/内容是否矛盾
4. 两人之间的信息传递是否合理

输出格式：
```markdown
## 交叉检测结果

### 一致性检查
| 检查项 | 结果 | 详情 |
|--------|------|------|
| 事件描述一致性 | ✓/✗ | |
| 信息提及一致性 | ✓/✗ | |
| 对话内容一致性 | ✓/✗ | |
| 信息传递合理性 | ✓/✗ | |

### 发现的问题
[详细描述每个问题]

### 修复建议
[具体可操作的修复方案]
```
```

## Prompt 4：逻辑漏洞检测

```
请检测以下剧本的**逻辑漏洞**：

【粘贴剧本概要 + 核心事件线】

检测维度：
1. **证据链完整性**：关键证据是否指向唯一凶手
2. **推理可能性**：玩家是否可以通过线索推理出真相
3. **死局检测**：是否存在无论如何都无法破局的情况
4. **角色动机**：每个角色的行为动机是否合理
5. **时间线可行性**：事件时间线是否物理可行

输出格式：
```markdown
## 逻辑漏洞检测

### 证据链分析
[分析每个关键证据]

### 推理路径
[列出可能的推理路径]

### 发现的问题
| 问题类型 | 描述 | 严重程度 | 修复建议 |
|----------|------|----------|----------|
| | | P0/P1/P2 | |

### 整体评估
[逻辑是否自洽]
```
```

```

3. 角色名/地名/物品名不一致
4. 明显的事实错误

```markdown
## 快速检测结果

### 发现问题
| # | 类型 | 位置 | 描述 |
|---|------|------|------|
| 1 | | | |

### 严重问题（需立即修复）
- ...

### 轻微问题（可后续修复）
- ...

### 结论
[是否可以继续使用此剧本]
```
