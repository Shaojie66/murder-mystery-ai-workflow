# Metrics Plan: murder-wizard Pro 落地页 A/B 测试

**产品阶段**: Pre-launch（上线前验证）
**追踪目标**: 实验 1（Fake Door）+ 实验 2（A/B 对比）
**追踪工具**: Plausible Analytics（轻量级、GDPR 合规、国内可访问）

---

## 核心追踪指标

### 实验 1：订阅付费 Fake Door

| 指标 | 说明 | 目标值 |
|------|------|--------|
| 落地页 PV | 页面访问量 | - |
| CTA 点击率 | 点击"立即预约"按钮的比例 | > 5% |
| 邮箱留资转化 | 点击后留下邮箱的比例 | > 20% |
| 弹窗打开率 | 模态框打开次数 / CTA 点击次数 | > 50% |

**成功标准**：点击率 > 5% 且留邮箱转化 > 20%

### 实验 2：A/B 测试

| 指标 | 说明 |
|------|------|
| 各版本 PV | `/landing-a` vs `/landing-b` 访问量 |
| 各版本 CTA 点击率 | 每个版本的 CTA 点击比例 |
| 各版本留邮箱转化率 | 每个版本的留邮箱比例 |
| 统计显著性 | 样本量满足后计算 p < 0.05 |

**成功标准**：B 版本转化率比 A 版本高 30% 以上

---

## 追踪事件定义

### 页面级事件

| 事件名 | 触发条件 | 携带参数 |
|--------|---------|---------|
| `page_view` | 页面加载完成 | `variant` (a/b/random), `referrer` |
| `scroll_50` | 滚动超过 50% | `variant`, `time_on_page` |
| `scroll_100` | 滚动到底部 | `variant`, `time_on_page` |

### 交互事件

| 事件名 | 触发条件 | 携带参数 |
|--------|---------|---------|
| `cta_click` | 点击"立即预约"按钮 | `variant`, `location` (hero/pricing/footer) |
| `modal_open` | 模态框打开 | `variant` |
| `email_submit` | 邮箱提交成功 | `variant` |
| `email_invalid` | 邮箱格式错误 | `variant` |
| `nav_click` | 点击导航链接 | `variant`, `target` (features/pricing/docs) |

---

## 漏斗分析

```
页面访问 (page_view)
    ↓
滚动 50% (scroll_50)         ← 跳出率在此检测
    ↓
CTA 点击 (cta_click)
    ↓
模态框打开 (modal_open)
    ↓
邮箱提交 (email_submit)
```

漏斗转化率：
- 整体转化率 = email_submit / page_view
- 各阶段转化率 = 下一事件 / 上级事件

## 统计显著性计算

当每个版本 PV ≥ 1000 后，计算：

```
转化率差值 = B_ctr - A_ctr
z-score = (B_ctr - A_ctr) / sqrt(p_pool * (1 - p_pool) * (1/n_a + 1/n_b))
p_pool = (A_conversions + B_conversions) / (n_a + n_b)

若 |z-score| > 1.96 → 95% 置信度统计显著
```

## 健康度指标

| 指标 | 健康范围 | 黄色预警 | 红色告警 |
|------|---------|---------|---------|
| 页面加载时间 | < 2s | 2-4s | > 4s |
| 跳出率 | < 60% | 60-80% | > 80% |
| 模态框错误率 | < 5% | 5-10% | > 10% |
| 重复提交率 | < 3% | 3-5% | > 5% |

## 工具选型

| 工具 | 用途 | 理由 |
|------|------|------|
| **Plausible Analytics** | 主追踪 | 轻量、GDPR 合规、国内可访问、无 Cookie |
| **Google Analytics 4** | 备选 | 功能全，但需 Cookie 同意 |
| **PostHog** | 进阶 | 自托管、支持 Product Analytics，但配置复杂 |

推荐先用 **Plausible**，符合隐私要求且配置简单。
